你现在是一个GUI智能体，接下来我将给你一个步骤，请你根据步骤描述和当前页面的截图，预测应该对什么UI元素做出什么动作。
并输出接下来一步动作的操作的元素类型和元素坐标，动作类型、内容：

1. 元素类型是指，每步操作的UI元素的类型。 

**ele_type space:**

| ele_type | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| icon     | 抽象图标类元素，即不含文字的图标                             |
| text     | 文本类元素，即含文字的图标                                 |
| box      | 输入框等元素                                                 |
| none     | 无元素类型，动作 get_info 不对应任何UI元素，故类型为none |


2. 动作类型及其内容： 

动作是指，agent每步应当执行的动作类型和内容。

**action space:**

| type     | content                                                      | 说明                                                         |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| click    | 所点击的元素内容（如果有），如点击文本类元素，content为点击的文本内容 | 点击                                                         |
| input    | 输入的文本                                                   | 输入文本                                                     |
| get_info | 获取到的信息                                                 | 获取信息操作，部分操作涉及到在界面获取信息                   |

3. 元素坐标：

元素坐标，即ele_loc，将横纵坐标分开输出，输出格式应该为：
    "ele_loc": {
        "x": <坐标x值>,
        "y": <坐标y值>
    }
如坐标点(100,200)应该输出为：
    "ele_loc": {
        "x": 100,
        "y": 200
    }

注意：
1. 输入框无需点击激活，你所看到的所有输入框都是已激活的，如果有需要在输入框输入文字的操作，只需直接执行input动作即可；
2. 元素类型，动作类型等请严格按照上述分类；
3. 步骤描述请直接输出对什么UI元素做什么动作。不要展示思考内容；

输出：
请严格按照以下格式输出：
{
    "ele_loc": {
        "x": ,
        "y": 
    },
